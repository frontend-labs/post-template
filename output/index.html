Actualizar repositorio forked a partir de las actualizaciones del repositorio original

17 enero, 2016Jan Sanchez

¿Qué significa hacerle Fork a un repositorio?

Hacerle fork a un repositorio significa copiarlo. Copiar un repositorio nos permite experimentar libremente sin afectar al proyecto original, obviamente porque nuestro fork es una copia, que se encontrará en nuestra cuenta de github, bitbucket o algún otro servicio como estos.

Por lo general, las copias se utilizan para de proponer cambios en el proyecto de otra persona u organización ó para utilizar el proyecto de otra persona como punto de partida para nuevas ideas.

Haciendo fork a un repositorio

Para poner en práctica este concepto vamos a hacerle fork al repositorio de relay, relay es un Framework Javascript creado por el equipo de desarrollo de Facebook.

Para llevar a cabo esto, primero necesitamos loguearnos en nuestra cuenta de github, si no tienes una cuenta de github, entonces antes debes crearla.

Una vez logueados, haremos un fork de este repositorio. Para esto nos dirigimos al repositorio de relay. En la página principal del repositorio de relay, podremos observar en la parte superior derecha el botón con la opción Fork. Lo presionamos y comenzará a realizar un Fork de ese repositorio hacia nuestra cuenta de Github.

En algunos casos en los que tengamos una cuenta personal asociada a alguna organización, Github nos pedirá que especifiquemos si el Fork se hará sobre nuestra cuenta personal o en alguna de nuestras cuentas de organización.

Una vez que le proceso haya terminado, habremos copiado ese repositorio a nuestra cuenta, así que luego de esto veremos un encabezado parecido al siguiente:

Como podemos ver Github nos indica que ahora tenemos en nuestra cuenta un nuevo repositorio llamado relay y vemos que debajo dice 'forked form facebook/relay' (copiado desde facebook/relay).

Trabajando con un repositorio Forked

Por lo general, cuando se hace Fork de un repositorio se realizan modificaciones sobre este repositorio para manipular la versión original sin causar problemas, es como tener un 'patio de juegos' para poder hacer lo que deseemos con el proyecto.

Pero a veces queremos hacer mejoras sobre ese proyecto y enviar nuestras mejoras al dueño original para que las implemente ó simplemente queremos seguir teniendo las actualizaciones del proyecto principal.

Esta vez nos enfocaremos en seguir teniendo las actualizaciones del proyecto principal, realizando los siguientes pasos:

Clonando nuestro repositorio Forked

Clonamos nuestro nuevo repositorio Forked, y nombraremos relay_forkeda la carpeta que almacenará nuestro proyecto.

git clone https://github.com/jansanchez/relay.git relay_forked

Una vez que terminamos de clonar nuestro nuevo repositorio, debemos indicarle a git que nuestro repositorio tiene una ubicación remota, en nuestro caso la url de la ubicación remota es: https://github.com/facebook/relay.git, y la agregamos a nuestro repositorio local escribiendo el siguiente comando:

git remote add upstream https://github.com/facebook/relay.git

Con esto le estamos diciendo a git: Que agregue la ubicación remota llamada upstream. Upstream es un término inglés que se puede interpretar como "principal" o "producción".

Luego de esto, ya estamos listos para poder actualizar nuestro repositorio desde la url remota principal, escribiendo el siguiente comando:

git fetch upstream

De esta forma obtenemos todas las ramas actualizadas desde esa ubicación remota.
